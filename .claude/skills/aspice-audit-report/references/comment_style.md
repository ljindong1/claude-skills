# Review Comment 작성 스타일 — 사람급 (본문 정독 심층 모드)

> ## ⚠ 가장 중요 — 검토결과서를 반드시 함께 읽어라
> 사람 코멘트의 **구체적 결함**(예: "인터페이스 테이블 ISLA 입력명 누락", "컴포넌트별 DFD/CFD 정합성 불일치")은 codebeamer가 아니라 **폴더에 있는 동료검토 검토결과서**(`*_검토결과서_*.xlsx`, `*_Review_Report_*.xls`, `*_동료검토_*.xls`)의 **'검토 결과'·'결함 내용' 시트**에 들어 있다. 산출물 본문(특히 codebeamer export로 Description이 빈 경우)만 읽고 "확인 필요"로 넘기면 사람 수준에 절대 도달하지 못한다.
> **각 점검대상 산출물마다, 짝이 되는 검토결과서를 찾아 결함(검증항목 ID·결함 내용·심각도·상태)을 읽고 해당 WP 문항에 인용하라.** 이게 사람급의 핵심 출처다.


사람 감사자의 코멘트를 모사하려면 **산출물 본문을 정독**하고 항목마다 아래 형식으로 쓴다.
결정론적 스크립트(judge())로는 불가능하다 — AI가 `extract_deliverable_text.py` 결과를 읽고 직접 작성한다.
토큰·시간이 많이 들므로, 정밀 점검이 필요한 핵심 산출물에 적용한다.

## 형식
`YYYY-MM-DD : {근거 위치} {확인 내용}. 다만 {미흡점/한계}. {권고/확인 필요}.`

## 필수 요소 (사람 코멘트에서 추출한 패턴)
1. **날짜 prefix**. 재검토하면 날짜를 누적한다("2026-05-08, 2026-05-24 : …").
2. **근거 위치를 정확히**: 절 번호('절 2.3.3'), 표/필드명('표 회의 체계', 'WBS 선후행 필드', '이해관계자 요구사항 Priority 필드'), URL, 교차문서('품질보증계획서 절 3.1').
3. **확인 동사**: "~을 확인함", "~로 정의됨".
4. **Pass여도 미흡점·한계를 기록**(Pass ≠ 완벽): "다만 AUTOSAR/MCAL 역할 범위가 명확히 문서화되어 있지 않음".
5. **권고/확인 필요**: "~ 정의 권고", "~ 직접 확인 필요".
6. **본문에 근거가 없으면 솔직히**: "상세 WBS는 PMS로 관리되어 계획서 본문 미수록 → PMS 직접 확인 필요".

## 금지
- 근거 위치 없는 일반 문구("문서 내 해당 항목 존재·충족 확인") 금지.
- 도메인 적정성("이 정도면 충분한가")을 단정하지 말 것 — 미흡점은 기록하되 최종 합격(Pass 확정)은 사람.
- 본문에 없는 내용을 있다고 쓰지 말 것(환각 금지).

## 사람 몫으로 남는 한계 (정직하게)
- 비트/정의 수준 내용 일관성 비교(추적성은 `trace_checker.py`로 일부 데이터화 가능).
- 인터뷰·메일 등 비문서 근거.
- 시스템(PMS·codebeamer·레드마인) 사실 — 커넥터 없으면 "확인 필요".

## 절차
1. `extract_deliverable_text.py <파일> --out text.md` 로 본문 전체(절 구조) 추출.
2. AI가 text.md를 읽고, 해당 산출물의 각 WP 문항에 위 형식으로 코멘트 작성 → `comments.json`.
3. `write_comments.py <결과.xlsx> comments.json` 로 결과 체크리스트에 주입.

## 산출물 두 종류 — 본문형 vs codebeamer export (중요)

사람급 코멘트 가능성은 산출물 형태로 갈린다.
- **본문형 docx**(프로젝트/형상/문제해결/변경/품질보증 계획서, 단위 테스트 계획서, 시스템 아키텍처 명세서 v1.0): 절·표에 실제 내용 있음 → **본문 정독으로 즉시 사람급 가능.**
- **codebeamer export**(요구사양서·아키텍처·시스템요구·상세설계 = "Export of ..."): 추출하면 모든 `Description:`이 비어 있음. export엔 **절 구조 + ID(SWR-/SWA-/SysRS-) + Status + Associations**만 남음. 실제 내용은 codebeamer에 있다.

### export 문서에서 export만으로 점검 가능한 것
1. 구조화/분류(절 구조), 2. 고유 식별(ID), 3. 추적 관계(Associations + TraceabilityBrowser → `trace_checker.py`), 4. 상태(Status; 대부분 Draft/In Review = 미승인).

### export만으로 불가 → codebeamer 필요
명확성·검증가능성·모순·일관성·HSI 반영·할당 적절성 등 내용 품질. → codebeamer 커넥터(`/api/v3/items`로 Description 수집, redmine 커넥터와 동일 패턴)로 내용을 끌어와야 사람급 작성 가능. 접근 없으면 "확인 필요"가 정직한 한계.

즉 사람급의 병목은 AI 작성력이 아니라 **내용 접근**이다. 내용에 닿으면 사람급, 닿지 못하면(export 껍데기) 누구도 불가.

## 전수·마커 원칙 (최종 게이트 연동)
- 점검대상(Applicable=Yes) 산출물은 **하나도 빠짐없이** 본문+검토결과서를 정독해 코멘트를 단다.
- 모든 코멘트 머리에 정독 마커를 붙인다: `(AI·본문정독)`/`(AI·결과서 정독)`/`(AI·검토결과서)`/`(AI·추적데이터)`/`(AI·구조확인)`/`(AI·확인필요)`. 이 마커가 있어야 `verify_comment_coverage.py`가 '심층'으로 인정한다.
- 근거가 폴더에 없으면 환각하지 말고 `(AI·확인필요)`로 사유를 적는다.
