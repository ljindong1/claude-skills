# aspice-audit-report — 릴리스 노트 & 로드맵 (evidence 브리핑 재정의)

작성일: 2026-07-09

## 0. 이 문서의 목적

스킬의 성격을 **"사람 답안 모사 판정 엔진"에서 "인간 auditor를 위한 1차 evidence 브리핑 + 점검 의도 초안 도구"로 재정의**하는 개편의 방향과 로드맵을 기록한다. 각 개편 항목의 배경·상충 규칙 화해·단계 계획을 담는다.

## 1. 재정의된 스킬 목적

이 스킬은 ASPICE PAM 4.0을 사내 전사 프로세스로 적용·이행하는지를 점검하는 활동을 보조한다. 최종 목적은 두 가지다.

1. **사전 evidence 정리** — 인간 auditor가 실제 audit을 수행하기 *전에*, 각 문항에 대한 evidence(산출물 내용·위치, process 이행 활동)를 1차로 정리해 수행 편의를 돕는다.
2. **점검 의도 초안 제공** — auditor가 체크리스트 문항을 오독·착각해 점검 의도를 벗어나지 않도록, 문항의 Intent(무엇을 확인하려는가) 초안을 함께 제공한다.

따라서 사람 작성본과의 일치율은 더 이상 북극성이 아니라, 있을 때만 쓰는 **보정 지표**다. 사람보다 먼저 도는 **블라인드 모드가 주 실행 모드**다.

## 2. 판정 원칙 (현행 확정)

- **Pass 기준** = 개별 문항의 Intent를 충족하는 **명확한 evidence가, 실제로 제공·정독한 산출물에 명시적으로 존재**하고 그것을 인용할 수 있을 때. Major/Minor 등급 구분은 현 단계에서 다루지 않는다.
- **표준 위치 + 프로젝트 특성 대체 모두 인정** — 예: 위험관리는 통상 프로젝트 계획서에 명시되지만, 프로젝트 특성에 따라 별도 파일·회의록으로 정리될 수 있다. 표준 위치가 아니어도 evidence가 명확하면 Pass.
- **대체 활동 인정(신규)** — ASPICE 문항은 매우 general하게 기술되어 조직이 adapt한다. 해당 활동이 다른 활동으로 수행되었고 그것이 evidence에 명시되면 Pass로 인정하되, **어떤 활동이 근거인지 review comment에 반드시 명시**한다.
  - 화해 원칙: 기존 `comment_style.md §7⒝`의 "유사 활동 대체 금지"는 폐기가 아니라 **정밀화**한다 → *Intent가 다른* 활동을 갖다 붙이는 것만 금지, *Intent가 동일함이 evidence로 확인되고 인용*되면 대체 Pass 허용.
- **Not-Pass 처리** — Pass가 아니면 review comment에 **어떤 산출물을 읽어야 하는지 지목**한다. 어느 산출물인지 명확하지 않으면 `확인 필요` 수준으로 기술한다.
- **정직성** — 제공된 산출물 어디에서도 명시적으로 확인되지 않으면 자동 Pass 금지. 환각 금지, 인용 필수.

## 3. 결정 기록 — Phase D(시스템 커넥터) 보류

**결정: 시스템 커넥터(PMS·codebeamer·레드마인 등 사내망 직접 접속) 구현을 잠정 보류한다.**

- **사유** — 사내망 시스템에 직접 접속하는 과정에서 운영 데이터의 **무결성을 훼손할 위험**이 있다.
- **대안(현행 유지)** — 점검 산출물은 **개발 서버에서 export한 복사본을 한 폴더에 모아 읽기 전용으로 정독**하는 방식을 표준으로 삼는다. 스킬은 이 폴더 밖으로 나가지 않는다.
- **결과(수용)** — 내용이 오직 사내 시스템에만 존재하는 항목(예: codebeamer export의 빈 Description, PMS 승인 로그)은 Phase D 없이는 `확인 필요 — <시스템>의 <항목> 확인 요망`으로 남는다. 이는 결함이 아니라 **목적 ①(auditor에게 어디를 볼지 알려주기)에 부합하는 정상 출력**이다.

### 3.1 향후 방향 (Phase D 재검토 시 전제)

Phase D를 다시 검토하게 될 경우 아래 전제를 지킨다.

- **읽기 전용 원칙** — 라이브 시스템에 어떤 쓰기도 하지 않는다. 무결성 보증이 없는 직접 접속은 금지.
- **선호 경로** — (a) 지금처럼 export 복사본을 폴더로 수집해 읽는 방식 유지, 또는 (b) 시스템의 읽기 전용 복제본/미러/공식 export API를 경유. 운영 원본 직접 조회는 최후.
- **적용 지표** — Process 점검대상의 상당수가 시스템 근거이므로, 재검토 시 이 영역이 자동 정리율을 끌어올리는 지렛대가 된다. 다만 무결성 리스크가 해소되기 전에는 착수하지 않는다.

## 4. 로드맵 (현행)

| 단계 | 내용 | 사내망 | 상태 |
|---|---|---|---|
| **A** | 스킬 성격·규칙 재정의 (SKILL 목적, `judge()` 자동 Pass 제거, `comment_style §7` 개정: Intent 라인 필수·대체활동 Pass 인용 규칙·not-Pass 산출물 지목) | 불필요 | 예정 |
| **B** | Intent 지식베이스 `intent_map.json` (문항→PAM 4.0 BP/outcome·초점·1차 위치·허용 대체 evidence), 사람 1회 검수 후 재사용 | 불필요 | 예정 |
| **C** | 다원 evidence 탐색 엔진 (`deliverable_map` 다대일 확장, 1차 없으면 폴더 전체에서 대체 근거 탐색, 게이트에 "Pass=인용 필수·not-Pass=다음 산출물 명시" 검사 추가) | 불필요 | 예정 |
| **D** | 시스템 커넥터로 `확인 필요`→Pass 전환 | 필요 | **보류(무결성 리스크, §3)** |

A~C는 export 복사본 폴더만으로 완결되며, 이것으로 목적 ①·②의 대부분을 충족한다. D는 리스크 해소 전까지 착수하지 않는다.
